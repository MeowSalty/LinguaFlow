翻译审校 ​

翻译审校功能帮助您确保翻译质量，支持逐段审校、内联编辑和批量操作。

审校流程 ​

text
翻译完成 → 逐段审校 → 编辑/批注 → 审批/拒绝 → 完成

翻译作业完成后的段落状态为「已翻译」，进入审校阶段。您也可以在创建翻译作业时开启 自动审批，跳过审校流程。

段落状态 ​

状态说明可转换为
待审校 (pending)等待翻译—
已翻译 (translated)AI 翻译完成，等待审校已批准、已驳回、已修改
已修改 (edited)人工编辑后的状态已批准、已驳回
已批准 (approved)审校通过—
已驳回 (rejected)审校不通过已批准、已修改、待审校（重新翻译）

状态流转

  • 审批操作可从「已翻译」「已修改」「已驳回」转换为「已批准」
  • 驳回操作可从「已翻译」「已修改」转换为「已驳回」
  • 已批准的段落不能直接驳回，需要先编辑修改

逐段审校 ​

翻译完成后，在工作区的段落列表中逐段查看和审校翻译结果。

界面示意（占位） · 段落审校
表格或对照视图展示原文 / 译文、状态标签、质量标记；支持行内编辑、批注与批量通过/拒绝。
待补截图：docs/public/images/review.png

审校界面 ​

每段显示以下信息：

  • 源文本 — 原始内容，可切换查看 HTML 源码
  • 翻译文本 — AI 翻译结果
  • 状态标签 — 带图标和颜色的状态标识（蓝色=待审校，绿色=已批准，红色=已驳回）
  • 质量指示 — 在译文单元格内显示质量检测结果（长度异常、重复翻译、未翻译、源语残留等）
  • 审校批注 — 审校者添加的备注信息，点击可展开查看

段落导航 ​

段落表格支持键盘上下导航：

  • ↑ / ↓ 键移动聚焦行
  • 自动滚动确保聚焦行不会被表格头部遮挡
  • 进入编辑时自动聚焦输入框

审校操作 ​

操作说明状态变化
✅ 审批确认翻译正确→ approved
❌ 拒绝标记翻译有问题→ rejected
✏️ 编辑内联编辑翻译内容→ edited

内联编辑 ​

点击编辑按钮可以直接修改翻译内容和添加批注：

  1. 点击段落的编辑按钮，输入框自动获得焦点
  2. 在编辑框中修改翻译文本
  3. 可选填写审校批注，记录修改原因
  4. 点击 保存 确认修改，或使用 「保存并下一个」 快捷进入下一段编辑

编辑后自动审批

编辑保存后的段落状态会变为「已修改」(edited)，仍需手动审批。

质量检测 ​

LinguaFlow 在翻译完成后自动检测译文中可能存在的问题，帮助您快速定位需要关注的段落。

检测规则 ​

检测类型（code）说明可 AI 裁决
长度异常（length_ratio）译文长度与原文相差过大（过短或过长）；可配置最小/最大长度比、按字符或按词计算✅ 软规则
重复翻译（duplicate）相邻段落译文完全相同❌ 硬规则
未翻译（untranslated）译文与原文完全一致，可能未被翻译❌ 硬规则
源语残留（source_residual）译文中夹带源语脚本片段（如假名、谚文、西里尔文、汉字残留等），按语言对自动匹配规则✅ 软规则

源语残留

检测器按源/目标语言对解析 Unicode 脚本规则：独立脚本（假名、谚文等）为强档；汉字在不同语言对间有准强 / 中等 / 弱档策略。专有名词、代码、刻意保留原文等场景可能产生误报，可配合 质量裁决 降噪。

质量指示 ​

检测到问题后，段落表格的质量指标列会显示对应的状态标记：

  • 绿色 — 无问题
  • 黄色 — 轻微问题（如源语残留、长度轻微偏差）
  • 红色 — 严重问题（如过长/过短超出阈值、重复、未翻译）

点击质量标记可查看详细诊断信息（含问题 code 与说明）。

质量检测配置 ​

在执行配置（Execution Profile）中调整质量检测参数：

yaml
qa:
  enabled: true # 是否启用质量检测
  length:
    enabled: true # 长度异常检测
    min_ratio: 0.5 # 最小长度比（译文/原文）
    max_ratio: 2.5 # 最大长度比
    unit: char # 计算方式：char（字符数）或 word（词数）
  repetition:
    enabled: true # 重复翻译检测
  untranslated:
    enabled: true # 未翻译检测

源语残留检测随质量检测引擎自动启用，无需单独开关（源语言为 auto 时不生效，需指定明确语言对）。

批注 ​

在审校过程中可以为每个段落添加批注，记录修改原因或翻译建议。

批注存储在段落的 review_comment 字段中，在段落列表中可以查看。

使用场景

  • 记录修改翻译的原因
  • 标记需要后续复查的术语
  • 为其他审校者提供参考意见

批量操作 ​

当翻译段落较多时，可以使用批量操作提高效率。

选择段落 ​

在段落列表中勾选需要操作的段落，底部会弹出浮动操作栏。

批量审批 ​

选择多个段落后，点击浮动操作栏的 通过 按钮，一次性审批所有选中的段落。

批量拒绝 ​

选择多个段落后，点击浮动操作栏的 拒绝 按钮，一次性拒绝所有选中的段落。

批量操作说明

  • 批量审批时，仅对状态为「已翻译」「已修改」「已驳回」的段落生效
  • 批量拒绝时，仅对状态为「已翻译」「已修改」的段落生效
  • 不满足条件的段落会被自动跳过

一键审批 ​

在资源详情页，可以使用 全部审批 功能，将该资源下所有符合条件的段落一次性审批通过。

  • 将所有「已翻译」「已修改」「已驳回」状态的段落转为「已批准」
  • 审批后自动清除这些段落的审校批注

重新翻译已驳回段落 ​

对于已驳回的段落，可以触发 重新翻译，将这些段落重置为「待审校」状态并重新进入翻译队列。

  • 所有「已驳回」状态的段落会被重置为「待审校」
  • 清除审校者信息和审校批注

自动审批 ​

创建翻译任务时，可以开启 自动审批 选项：

  • 开启后，翻译完成的段落会直接标记为「已批准」，无需手动审校
  • 适用于对翻译质量要求不高的场景或初稿翻译

在任务创建面板中，勾选「翻译完成后自动审批」即可启用。

审校进度 ​

工作区顶部的进度条显示翻译和审校进度：

  • 蓝色进度条 — 已翻译段落占比（包含已翻译、已修改、已批准）
  • 绿色进度条 — 已审批段落占比（仅已批准）

段落分组也会显示每组的已批准数量和占比。

搜索与筛选 ​

在段落列表中可以搜索和筛选，快速定位需要审校的段落。

搜索 ​

在段落列表顶部的搜索框中输入关键词，可以同时搜索原文和译文内容。搜索为模糊匹配，输入任意关键词即可筛选出包含该内容的段落。

搜索与筛选可叠加

搜索可以与状态筛选同时使用。例如，先筛选「已驳回」状态，再搜索特定术语，快速找到需要修改的段落。

按状态筛选 ​

在段落列表中可以按状态筛选：

  • 筛选「已翻译」— 查看待审校的段落
  • 筛选「已驳回」— 查看需要重新处理的段落
  • 筛选「已批准」— 查看已通过的段落
  • 筛选「已修改」— 查看人工编辑过的段落

按质量问题筛选 ​

段落列表支持按质量问题过滤，快速定位需关注的段落：

筛选维度选项
有无问题有问题 / 无问题
严重程度error / warning
问题类型untranslated / length_ratio / duplicate / source_residual

可与状态筛选、关键词搜索叠加。例如：先筛「有问题」+ source_residual，再结合状态「已翻译」，集中处理源语残留。

配合 AI 质量裁决

规则质检偏敏感，难免误报。可在执行计划中增加 质量裁决（adjudicate）轮次，让 AI 对 source_residual、length_ratio 等软规则逐条复核，剔除明确误报后再人工审校。详见 流水线与原理 · 质量裁决。

最佳实践 ​

审校建议

  1. 分批审校 — 按章节或段落分批审校，避免疲劳
  2. 关注术语 — 重点检查专业术语的翻译是否一致
  3. 保持上下文 — 审校时注意段落之间的连贯性
  4. 记录批注 — 为重要的修改添加批注，方便后续参考
  5. 善用筛选 — 按状态与质量问题筛选，优先处理有问题的段落
  6. 批量操作 — 段落较多时使用批量审批/拒绝提高效率
  7. 裁决降噪 — 启用质量裁决轮次，减少源语残留、长度异常的误报干扰
