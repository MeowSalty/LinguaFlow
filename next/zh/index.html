把文档、字幕、电子书和配置文件交给 AI，在本地完成批量翻译与审校——开源、单二进制、开箱即用。

一次跑通整条链路

上传源文件 → 配置 AI → 批量翻译 → 在线审校导出。Web 可视化与 CLI 脚本两种用法，同一套翻译能力。

📄

多格式，结构不丢

Markdown、HTML、DOCX、EPUB、字幕（SRT/VTT/ASS）、JSON/YAML/TOML、纯文本。按格式切段与回写，代码块与链接自动保护。

📚

术语一致，质量可控

项目术语表、自动提取与精简；长度异常、未译、源语残留等质检；可选 AI 裁决降噪后再人工审校。

🤖

你的模型，你的网关

OpenAI / Anthropic / Gemini，以及 Azure、Ollama、LM Studio 等兼容接口。自定义提示词与多轮执行计划。

💻

本地优先，隐私可控

双击二进制即可启动本地模式，数据落在本机 SQLite。无需账号、无需把全文交给第三方 SaaS 平台。

⚡

批量与增量

整目录批量处理，文件内段落分批并发；实时进度与 ETA；增量更新保留已有译文，只处理新增或变更内容。

适合做什么 ​

场景你能得到
文档站 / README整目录 Markdown 批量译成多语言，格式与代码块尽量保持原样
字幕SRT / VTT / ASS 保留时间码，专注译文本
电子书EPUB 按章节浏览与翻译，导出仍是 EPUB
配置与 i18nJSON / YAML / TOML 按字符串叶子节点翻译，结构路径可回写
Word 文稿DOCX 解析主文档段落，便于校对与导出

一行安装 ​

bash
# 从 Releases 下载后：
# https://github.com/MeowSalty/LinguaFlow/releases
./linguaflow          # Linux / macOS，自动打开浏览器
# 或双击 linguaflow.exe（Windows）
# 默认：本地模式 http://127.0.0.1:18080
bash
docker pull ghcr.io/meowsalty/linguaflow:latest
docker run -p 8080:8080 ghcr.io/meowsalty/linguaflow:latest
# 访问 http://localhost:8080（容器默认服务器模式，见安装文档）
bash
linguaflow init
# 编辑 linguaflow.yaml，填入 API Key
linguaflow translate -i README.md -o README_zh.md --to zh

为什么是 LinguaFlow ​

  • 工作台，而不只是一次对话 — 项目、资源、作业、术语、审校状态可沉淀，适合反复迭代的本地化任务
  • 格式感知 — 不是整文件糊进聊天窗口，而是按文档结构切段、保护不可译内容、再精准回写
  • 可编排 — 执行计划支持翻译、术语提取、质量裁决等多轮流水线，按场景组合
  • 开源可自托管 — AGPL v3；个人与本地场景以单二进制 + SQLite 为主路径
