# 项目管理

项目是组织一次翻译工作的基本单位：语言方向、源文件（资源）、术语表与翻译作业都挂在项目下。

尚未跑通第一次翻译？请先看 [快速开始 · Web](/zh/guide/getting-started)。名词解释见 [核心概念](/zh/guide/concepts)。

## 创建项目

1. 打开 **项目** 页，点击 **新建项目**
2. 填写项目信息：
   - **项目名称** — 用于标识项目（最多 80 个字符）
   - **源语言** — 源文件语言，可用 `auto` 自动识别
   - **目标语言** — 要译成的语言
   - **启用术语表** — 是否在翻译时应用术语规则
3. 点击 **创建**，再点进项目卡片进入工作区

::: tip 提示
创建后可随时改名称、语言与术语表开关。本地模式下你就是唯一用户，无需关心组织归属。
:::

### 支持的语言（节选）

| 代码      | 语言                   |
| --------- | ---------------------- |
| `auto`    | 自动识别（仅限源语言） |
| `zh-Hans` | 简体中文               |
| `zh-Hant` | 繁体中文               |
| `en-US`   | 英语（美国）           |
| `en-GB`   | 英语（英国）           |
| `ja`      | 日语                   |
| `ko`      | 韩语                   |
| `fr`      | 法语                   |
| `de`      | 德语                   |
| `es`      | 西班牙语               |

界面中通常提供更完整的语言列表。

### 项目归属（服务器模式 · 预览）

::: warning 预览
组织与多用户归属依赖服务器模式，该模式仍在完善。个人本地使用时只需个人项目即可。
:::

- **个人项目** — 归属当前用户  
- **组织项目** — 归属组织（需相应权限）  

## 项目列表

首页展示所有项目列表，支持以下功能：

- **搜索过滤** — 按项目名称或语言代码搜索
- **统计面板** — 显示全部项目数和语言方向数
- **排序** — 按更新时间降序排列
- **快捷操作** — 右键菜单可查看详情、编辑、查看任务、管理术语表或删除项目

## 项目工作区

进入项目后，顶部显示项目基本信息（名称、语言方向、更新时间）和统计指标栏：

> **界面示意（占位）** · 项目工作区  
> 顶部：项目名、语言方向、资源/段落/已译/作业等统计；下方四个标签：**资源 · 段落 · 作业 · 术语表**。  
> *待补截图：`docs/public/images/workspace.png`*

| 指标       | 说明                 |
| ---------- | -------------------- |
| 资源数     | 项目中的文件总数     |
| 段落数     | 待翻译的内容段总数   |
| 已翻译     | 已完成翻译的段落数   |
| 已审核     | 已通过审核的段落数   |
| 运行中作业 | 正在执行的翻译作业数 |

工作区包含四个标签页：

1. **资源文件** — 文件管理（详见下方）
2. **段落编辑** — 翻译内容查看与编辑
3. **翻译作业** — 翻译作业管理（详见下方）
4. **术语表** — 项目术语管理

## 管理翻译资源

### 上传文件

支持以下方式上传文件：

- **拖拽上传** — 将文件拖拽到上传区域
- **点击上传** — 点击上传按钮选择文件
- **批量上传** — 一次最多上传 50 个文件

::: tip 提示
上传前会自动进行冲突检测。如果同名文件已存在，可选择替换或增量更新。
:::

### 支持的文件格式

| 格式     | 扩展名                     | 说明            |
| -------- | -------------------------- | --------------- |
| Markdown | `.md`, `.markdown`, `.mdx` | 通用文档格式    |
| HTML     | `.html`, `.htm`            | 标记语言文档    |
| DOCX     | `.docx`                    | Word 文档       |
| EPUB     | `.epub`                    | 电子书格式      |
| SRT      | `.srt`                     | 字幕格式        |
| VTT      | `.vtt`                     | Web 字幕格式    |
| ASS      | `.ass`                     | 高级字幕格式    |
| 纯文本   | `.txt`                     | 纯文本文件      |
| JSON     | `.json`                    | JSON 结构化格式 |
| YAML     | `.yaml`, `.yml`            | YAML 结构化格式 |
| TOML     | `.toml`                    | TOML 结构化格式 |

详见 [格式支持](/zh/guide/formats)。

### 资源管理

上传后的资源可以进行以下操作：

- **查看** — 浏览源文件内容
- **翻译** — 启动翻译作业
- **替换** — 上传新文件替换已有资源
- **增量更新** — 在保留已有翻译的基础上更新源文件
- **下载** — 下载原始文件或翻译结果文件
- **删除** — 删除资源及其所有段落和翻译

### EPUB 特殊支持

EPUB 格式文件具有专用的虚拟目录结构和章节导航，支持按章节选择翻译。

## 翻译作业

### 创建翻译作业

1. 在项目中选择需要翻译的资源（或资源中的特定段落/章节）
2. 选择翻译执行计划
3. 配置作业参数：
   - **过滤配置** — 控制对已有翻译的处理方式
   - **自动审核** — 翻译完成后是否自动标记为已审核
4. 点击 **开始翻译**

### 过滤配置

创建作业时可配置段落过滤，控制哪些段落需要重新翻译：

| 过滤选项   | 说明                         |
| ---------- | ---------------------------- |
| 跳过已批准 | 跳过已审批通过的段落（默认） |
| 覆盖未批准 | 覆盖所有未审批通过的段落     |
| 覆盖全部   | 覆盖所有段落，无论状态       |

过滤配置可在执行计划中预设默认值，创建作业时按需覆盖。

### 作业状态

| 状态   | 说明                           |
| ------ | ------------------------------ |
| 等待中 | 作业已创建，等待执行           |
| 运行中 | 正在进行翻译                   |
| 已完成 | 翻译完成                       |
| 失败   | 翻译失败，可查看错误信息或重试 |
| 已取消 | 用户手动取消的作业             |

### 进度追踪

翻译过程中通过 SSE 实时推送进度信息（支持断线重连、事件持久化）：

- **翻译进度** — 已完成段数 / 总段数
- **预计剩余时间 (ETA)**
- **当前翻译速度**
- **事件序号（seq）** — 保证事件顺序，重连后可精准恢复丢失的事件

### 作业操作

- **取消** — 取消正在运行的作业
- **重试** — 对失败的作业重新执行翻译

## 项目设置

### 语言设置

- 修改源语言和目标语言
- 切换源语言自动识别

### 术语表

将术语表关联到项目，翻译时自动应用术语规则。启用后，翻译引擎会优先使用术语表中的翻译结果。

详见 [术语表管理](/zh/guide/glossary)。

## 项目删除

删除项目时会**级联删除**以下所有关联数据：

- 翻译作业及其关联关系
- 资源文件及其段落
- 术语表条目
- 翻译记忆条目
- 同步任务记录
- 活动日志和使用记录

::: danger 警告
项目删除操作不可撤销，请谨慎操作。
:::
