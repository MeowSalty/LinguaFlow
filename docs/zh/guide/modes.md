# 使用模式

LinguaFlow 提供两种运行模式，分别适用于不同的使用场景。

## 本地模式

本地模式适用于个人使用，无需登录认证，数据存储在本地。后端会自动创建一个名为 `local` 的本地用户，所有请求自动以该用户身份通过认证，完全跳过 JWT 验证流程。

### 启动方式

```bash
linguaflow local
```

或者直接双击运行 `linguaflow`（Windows）/ `./linguaflow`（Linux/macOS），程序通过 `mousetrap` 检测到双击启动后，自动以本地模式运行。

启动后会自动打开浏览器访问 `http://127.0.0.1:18080`。

### 特点

- **无需登录** — 启动即可使用，前端自动跳过登录和注册页面
- **自动打开浏览器** — 默认访问 `http://127.0.0.1:18080`，可通过 `--no-browser` 禁用
- **嵌入式前端** — 前端静态资源打包在后端二进制中，无需单独部署
- **本地数据存储** — SQLite 数据库，数据目录为系统用户配置目录下的 `LinguaFlow` 文件夹
- **端口冲突处理** — 如果端口被占用，自动递增端口号，最多尝试 10 次
- **CORS 限制** — 仅允许来自 `127.0.0.1` 和 `localhost` 的请求

### 配置选项

| 参数           | 描述             | 默认值                             |
| -------------- | ---------------- | ---------------------------------- |
| `--port`       | 监听端口         | `18080`                            |
| `--host`       | 监听地址         | `127.0.0.1`                        |
| `--db`         | 数据库路径       | `UserConfigDir/LinguaFlow/data.db` |
| `--no-browser` | 不自动打开浏览器 | `false`                            |

::: tip 数据目录说明
`UserConfigDir` 因操作系统而异：

- Windows: `%AppData%`（如 `C:\Users\<用户名>\AppData\Roaming`）
- macOS: `~/Library/Application Support`
- Linux: `~/.config`（遵循 `XDG_CONFIG_HOME`）
  :::

## 服务器模式

服务器模式适用于团队协作，支持多用户、认证和权限管理。

### 启动方式

```bash
linguaflow serve
```

### 特点

- **多租户** — 支持多用户注册和登录
- **JWT 认证** — 基于 Token 的身份验证（Access Token 有效期 15 分钟，Refresh Token 有效期 30 天）
- **RBAC 权限** — 基于角色的访问控制
- **组织管理** — 支持组织和团队管理
- **网络访问** — 默认监听 `0.0.0.0`，支持局域网/公网访问
- **用户注册** — 默认开放注册，首个注册用户自动成为管理员
- **前端独立部署** — 不提供前端静态资源，需单独部署前端应用

### 配置选项

| 参数             | 描述           | 默认值           |
| ---------------- | -------------- | ---------------- |
| `--port`         | 监听端口       | `8080`           |
| `--host`         | 监听地址       | `0.0.0.0`        |
| `--db`           | 数据库路径     | `./data/data.db` |
| `--auto-migrate` | 自动迁移数据库 | `true`           |

### 管理员配置

服务器模式下，可通过环境变量配置初始管理员账户：

```bash
export LINGUAFLOW_ADMIN_USERNAME=admin
export LINGUAFLOW_ADMIN_PASSWORD=your-secure-password
linguaflow serve
```

如果未设置环境变量，首个注册的用户将自动成为管理员（取决于配置中 `registration.auto_admin` 的设置）。

::: warning 安全提示
服务器模式下，请确保：

- 使用强密码
- 修改默认的 JWT Secret（配置项 `server.jwt_secret`）
- 在生产环境中配置 HTTPS（通过反向代理）
- 限制监听地址为内网或配置防火墙
  :::

## 模式对比

| 特性           | 本地模式                   | 服务器模式         |
| -------------- | -------------------------- | ------------------ |
| CLI 命令       | `linguaflow local`         | `linguaflow serve` |
| 用户认证       | 自动认证（跳过 JWT）       | JWT Token 认证     |
| 多用户         | 单用户（`local` 用户）     | 多租户             |
| 用户注册       | 不支持                     | 默认开放           |
| 默认端口       | `18080`                    | `8080`             |
| 默认监听       | `127.0.0.1`                | `0.0.0.0`          |
| 数据目录       | `UserConfigDir/LinguaFlow` | `./data`           |
| 前端资源       | 嵌入在二进制中             | 需单独部署         |
| CORS 策略      | 仅 localhost               | 可配置（默认 `*`） |
| 自动打开浏览器 | 是                         | 否                 |
| 双击启动       | 自动进入本地模式           | 不支持             |
| 适用场景       | 个人使用                   | 团队协作           |

## 前端模式适配

前端应用会通过 `/api/v1/mode` 接口（无需认证）自动检测当前运行模式，并据此调整行为：

- **本地模式**：跳过登录/注册页面，直接进入主界面；用户菜单仅显示"连接远程服务"选项；顶部显示绿色 "本地模式" 标签
- **服务器模式**：需要登录后才能访问；用户菜单显示"切换服务"和"退出登录"选项
