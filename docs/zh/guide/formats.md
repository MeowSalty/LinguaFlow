# 格式支持

LinguaFlow 支持多种文件格式的翻译，每种格式有不同的处理方式。

## 支持的格式

| 格式     | 扩展名                     | 说明                               |
| -------- | -------------------------- | ---------------------------------- |
| Markdown | `.md`, `.markdown`, `.mdx` | 通用文档格式                       |
| HTML     | `.html`, `.htm`            | 标记语言文档                       |
| DOCX     | `.docx`                    | Word 文档                          |
| EPUB     | `.epub`                    | 电子书格式                         |
| SRT      | `.srt`                     | SubRip 字幕格式                    |
| VTT      | `.vtt`                     | WebVTT 字幕格式                    |
| ASS      | `.ass`                     | Advanced SubStation Alpha 字幕格式 |
| 纯文本   | `.txt`                     | 纯文本文件                         |
| JSON     | `.json`                    | JSON 结构化格式                    |
| YAML     | `.yaml`, `.yml`            | YAML 结构化格式                    |
| TOML     | `.toml`                    | TOML 结构化格式                    |

## Markdown

Markdown 是 LinguaFlow 支持最完整的格式。

### Markdown 特性

- **格式保留** — 保留标题、列表、表格等格式
- **AST 解析** — 基于 goldmark AST 的叶子块解析，精准定位可翻译单元
- **代码块保护** — 自动识别并保护代码块不被翻译
- **链接保护** — 保护 URL 和链接文本
- **图片保留** — 保留图片链接和 alt 文本

### Markdown 注意事项

::: warning

- HTML 标签内的文本可能会被翻译
- 复杂的嵌套格式可能影响翻译质量
  :::

## HTML

HTML 格式保留标签结构，仅翻译文本节点。

### HTML 特性

- **标签保护** — 保护 HTML/XML 标签及其属性，仅翻译可读文本节点
- **结构保留** — 渲染时按字节区间回写译文，保持文档结构
- **按块切段** — 按块级元素（`p` / `h1-h6` / `li` / `td` 等）与 `<title>` 切段
- **大小限制** — 支持最大 16 MB 的 HTML 文件

::: info 暂不翻译的内容
`script` / `style` / `code` / `pre` 内的内容以及可翻译属性（如 `alt`、`title`）在当前版本中原样保留，不进入翻译段。
:::

::: tip
含 `<ruby>` 注音的 HTML 可配合执行配置中的 Ruby 注音处理（详见 [高级功能](/zh/guide/advanced)）。
:::

## DOCX

DOCX（`.docx`）解析器直接处理 Word 文档的内部 ZIP + XML 结构，无需依赖 Microsoft Word 或其它外部工具。

### DOCX 特性

- **ZIP 包结构保留** — 原样保留原始包内文件及压缩方式，仅重写 `word/document.xml`
- **段落级定位** — 按 `element_path` 定位 `<w:p>` 段落，保留段落属性与样式外壳
- **内联语义** — 粗体、斜体等内联语义以 HTML 表示，便于审校界面预览
- **大小保护** — 整包上限 100 MB，并对解压后的单个条目设有上限，防止解压炸弹攻击

### DOCX 注意事项

::: warning
页眉页脚、批注、SmartArt、图表等非主文档段落内容可能不被翻译；建议翻译后用 Word 打开校对排版。
:::

## EPUB 电子书

EPUB 格式的翻译支持章节级别的管理。

### EPUB 特性

- **章节浏览** — 按章节浏览电子书内容
- **逐章翻译** — 按章节进行翻译
- **进度追踪** — 按章节追踪翻译进度
- **格式保留** — 保留 HTML 内容和样式
- **DRM 检测** — 自动检测并拒绝 DRM 保护的文件
- **最大文件限制** — 支持最大 100MB 的 EPUB 文件

### 翻译流程

1. 上传 EPUB 文件
2. 系统自动解析章节结构
3. 选择章节进行翻译
4. 查看翻译进度
5. 导出翻译后的 EPUB

::: tip EPUB 翻译建议

- 建议按章节顺序翻译，保持上下文连贯
- 使用术语表确保术语一致性
  :::

## SRT 字幕

SubRip 字幕格式，常用于视频字幕。

### SRT 格式示例

```srt
1
00:00:01,000 --> 00:00:04,000
Hello, welcome to the tutorial.

2
00:00:05,000 --> 00:00:08,000
Today we will learn about LinguaFlow.
```

### SRT 翻译特性

- **时间码保留** — 保留原始时间码
- **序号保留** — 保留字幕序号
- **格式保留** — 保留 SRT 格式结构

## VTT 字幕

WebVTT 字幕格式，常用于 Web 视频。

### VTT 格式示例

```vtt
WEBVTT

00:00:01.000 --> 00:00:04.000
Hello, welcome to the tutorial.

00:00:05.000 --> 00:00:08.000
Today we will learn about LinguaFlow.
```

### VTT 翻译特性

- **时间码保留** — 保留原始时间码
- **格式保留** — 保留 VTT 格式结构

## ASS 字幕

Advanced SubStation Alpha 字幕格式，支持更丰富的样式。

### ASS 翻译特性

- **样式保留** — 保留 ASS 样式定义
- **时间码保留** — 保留原始时间码
- **事件保留** — 保留 ASS 事件结构

::: warning ASS 格式注意
ASS 格式较为复杂，翻译时请确保样式定义不被修改。
:::

## 纯文本

纯文本格式的翻译是最简单的场景。

### 纯文本翻译特性

- **逐段翻译** — 按空行分段进行翻译
- **编码支持** — 支持 UTF-8 编码
- **XUnity 格式支持** — 自动检测 XUnity AutoTranslator 格式（key=value）

## JSON

JSON 结构化格式，适用于配置文件、本地化文件等。

### JSON 翻译特性

- **递归解析** — 递归收集所有字符串叶子节点
- **路径保留** — 记录每个字符串的 JSON 路径
- **格式保留** — 保留 JSON 结构和缩进

## YAML

YAML 结构化格式，适用于配置文件、文档等。

### YAML 翻译特性

- **递归解析** — 递归收集所有字符串叶子节点
- **路径保留** — 记录每个字符串的 YAML 路径
- **格式保留** — 保留 YAML 结构

::: warning YAML 格式限制
YAML 注释在翻译过程中可能无法保留。
:::

## TOML

TOML 结构化格式，适用于配置文件、本地化文件等。

### TOML 翻译特性

- **递归解析** — 递归收集所有字符串叶子节点
- **路径保留** — 记录每个字符串的 TOML 路径
- **格式保留** — 保留 TOML 结构
- **日期时间保留** — TOML 的日期时间类型（`time.Time`）为非字符串叶子，不进入翻译段，渲染时原样保留

::: warning TOML 格式限制
TOML 注释在翻译过程中可能无法保留（通用反序列化机制限制）。JSON、YAML、TOML 三者共享同一解析器，浏览器/解析器会通过尝试解析来区分格式，避免误判。
:::

## 格式选择建议

::: tip 选择建议

- **文档翻译** — 使用 Markdown 格式
- **HTML 内容** — 使用 HTML 格式
- **Word 文档** — 使用 DOCX 格式
- **电子书翻译** — 使用 EPUB 格式
- **视频字幕** — 使用 SRT 或 VTT 格式
- **配置文件** — 使用 JSON / YAML / TOML 格式
- **简单文本** — 使用纯文本格式
  :::
