# 格式支持

LinguaFlow 支持多种文件格式的翻译，每种格式有不同的处理方式。

## 支持的格式

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| Markdown | `.md`, `.markdown`, `.mdx` | 通用文档格式 |
| EPUB | `.epub` | 电子书格式 |
| SRT | `.srt` | SubRip 字幕格式 |
| VTT | `.vtt` | WebVTT 字幕格式 |
| ASS | `.ass` | Advanced SubStation Alpha 字幕格式 |
| 纯文本 | `.txt` | 纯文本文件 |
| JSON | `.json` | JSON 结构化格式 |
| YAML | `.yaml`, `.yml` | YAML 结构化格式 |
| TOML | `.toml` | TOML 结构化格式（暂未实现） |

## Markdown

Markdown 是 LinguaFlow 支持最完整的格式。

### 特性

- **格式保留** — 保留标题、列表、表格等格式
- **代码块保护** — 自动识别并保护代码块不被翻译
- **链接保护** — 保护 URL 和链接文本
- **图片保留** — 保留图片链接和 alt 文本

### 注意事项

::: warning
- HTML 标签内的文本可能会被翻译
- 复杂的嵌套格式可能影响翻译质量
:::

## EPUB 电子书

EPUB 格式的翻译支持章节级别的管理。

### 特性

- **章节浏览** — 按章节浏览电子书内容
- **逐章翻译** — 按章节进行翻译
- **进度追踪** — 按章节追踪翻译进度
- **格式保留** — 保留 HTML 内容和样式
- **DRM 检测** — 自动检测并拒绝 DRM 保护的文件
- **最大文件限制** — 支持最大 100MB 的 EPUB 文件

### 翻译流程

1. 上传 EPUB 文件
2. 系统自动解析章节结构
3. 选择章节进行翻译
4. 查看翻译进度
5. 导出翻译后的 EPUB

::: tip EPUB 翻译建议
- 建议按章节顺序翻译，保持上下文连贯
- 使用术语表确保术语一致性
:::

## SRT 字幕

SubRip 字幕格式，常用于视频字幕。

### 格式示例

```srt
1
00:00:01,000 --> 00:00:04,000
Hello, welcome to the tutorial.

2
00:00:05,000 --> 00:00:08,000
Today we will learn about LinguaFlow.
```

### 翻译特性

- **时间码保留** — 保留原始时间码
- **序号保留** — 保留字幕序号
- **格式保留** — 保留 SRT 格式结构

## VTT 字幕

WebVTT 字幕格式，常用于 Web 视频。

### 格式示例

```vtt
WEBVTT

00:00:01.000 --> 00:00:04.000
Hello, welcome to the tutorial.

00:00:05.000 --> 00:00:08.000
Today we will learn about LinguaFlow.
```

### 翻译特性

- **时间码保留** — 保留原始时间码
- **格式保留** — 保留 VTT 格式结构

## ASS 字幕

Advanced SubStation Alpha 字幕格式，支持更丰富的样式。

### 翻译特性

- **样式保留** — 保留 ASS 样式定义
- **时间码保留** — 保留原始时间码
- **事件保留** — 保留 ASS 事件结构

::: warning ASS 格式注意
ASS 格式较为复杂，翻译时请确保样式定义不被修改。
:::

## 纯文本

纯文本格式的翻译是最简单的场景。

### 翻译特性

- **逐段翻译** — 按空行分段进行翻译
- **编码支持** — 支持 UTF-8 编码
- **XUnity 格式支持** — 自动检测 XUnity AutoTranslator 格式（key=value）

## JSON

JSON 结构化格式，适用于配置文件、本地化文件等。

### 翻译特性

- **递归解析** — 递归收集所有字符串叶子节点
- **路径保留** — 记录每个字符串的 JSON 路径
- **格式保留** — 保留 JSON 结构和缩进

## YAML

YAML 结构化格式，适用于配置文件、文档等。

### 翻译特性

- **递归解析** — 递归收集所有字符串叶子节点
- **路径保留** — 记录每个字符串的 YAML 路径
- **格式保留** — 保留 YAML 结构

::: warning YAML 格式限制
YAML 注释在翻译过程中可能无法保留。
:::

## TOML

TOML 结构化格式。

::: warning TOML 格式注意
TOML 格式暂未实现，目前仅支持 JSON 和 YAML。
:::

## 格式选择建议

::: tip 选择建议
- **文档翻译** — 使用 Markdown 格式
- **电子书翻译** — 使用 EPUB 格式
- **视频字幕** — 使用 SRT 或 VTT 格式
- **配置文件** — 使用 JSON 或 YAML 格式
- **简单文本** — 使用纯文本格式
:::
